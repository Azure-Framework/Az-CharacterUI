## Overview

This document summarizes the main changes, fixes, goals, and follow-up items discussed for **Az-CharacterUI** during the past week.

---

<details open>
<summary><strong>Main Work Covered</strong></summary>

## Main Work Covered

- Integrated Az-First Join behavior into Az-CharacterUI.
- Added a `Config.UseFirstJoin` toggle so first-join flow can be enabled or disabled cleanly.
- Adjusted the flow order so first-join appears after `Config.SpawnDeathScreen` instead of interfering with the death/spawn process.
- Continued config cleanup so CharacterUI behaves more like a normal FiveM config instead of an overly defensive or auto-generated style.

</details>

<details open>
<summary><strong>Notable Requested Behavior</strong></summary>

## Notable Requested Behavior

- First-join sequence should feel like part of CharacterUI, not like a separate bolted-on system.
- The config should be readable and easy to edit manually.
- Preview-related options, including mugshot behavior, needed to exist in a more standard config format.

</details>

<details open>
<summary><strong>Config Cleanup Direction</strong></summary>

## Config Cleanup Direction

- Replace verbose defensive config initialization with regular editable config values.
- Ensure settings like mugshot, department text, props, and hand bone are clearly present and editable.
- Avoid patterns that make the config feel autogenerated or hard to maintain.

</details>
